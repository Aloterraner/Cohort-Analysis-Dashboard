User Guide

Introduction

Welcome to Cohort Analysis Dashboard, or CAD for short.

CAD extends on the UI Framework for PM4PY to allow the applications of methods of Cohort Analysis to Process Data with a focus on data derived from the Medical Domain. CAD's goal is to create a Dashboard for users working with process data, providing them with informative views on the process and creating new insights for the process owner. CADs aim is to allow users to analyse processes by creating event groups and view their relation to the process by means of learning about the number of concurrently running events and the time intervals each group of events is happening in.

Usage Flow

There are three main sections:

  1. Log Management - dedicated for managing new and existing logs
  2. Cohort Analysis - dedicated for analysing log using Cohort Analysis techniques
  3. (Perspective) Views - dedicated for showing quick log overviews for doctors

The starting point of any session should be the ‘Log Management’ section, as you will need to upload and/or select the corresponding log you’re interested in. Once a log is selected, there are no limitations which further section should be used.

Glossary

# Key Description
1 CAD Cohort Analysis Dashboard
2 Cohort A group of individuals having a statistical factor (such as age or class membership) in common in a demographic study. In the scope of CADs a cohort of traces is based on them containing a certain type of activity happening in it.
3 GUI Graphical User Interface
4 XES eXtensible Event Stream: Standard is to standardize a language to transport, store, and exchange (possibly large volumes of) event data (e.g., for process mining)
5 Activity / Event An activity is an action that is performed by a resource in the scope of a process observed in an organization
6 Trace A trace is a temporal-ordered sequence of activities, representing a chain of activities performed in an organization
7 Log A collection of traces. Generally assumed to be well-formed in a given standard for process mining, such as XES.

Data Management

Importing Log

The starting point of using CAD is to upload a log. For now we are supporting only logs in either CSV or XES formats.

In order to import a new log into CAD you need to:

  1. Navigate to Log Management section
  2. Choose a file from your local machine
  3. Click on upload Event Log button

Selecting log for further analysis

Once a log is imported, you might want to select and use it for further Analysis.

This can be achieved by clicking on the desired log in the logs list, and clicking the ‘Select’ button

This will open a new ‘Selection View’, where you will have to let CAD know about some log specific information, such as configuring the ‘Case Id’ or specifying whether the log is a ‘Lifecycle Interval’ log.

Once the log attributes are selected, you can click on the ‘Select’ button, and this will configure the log as the current log. You can see the currently selected log in the ‘Log Management’ section:

Deleting Log

In case a log is not needed anymore, you can easily delete it by selecting it from the log list in the ‘Log Management’ section, and clicking the ‘Delete’ button.

Exporting Log

Any log imported to CAD can also be exported by navigating to the ‘Log Management’ section, and clicking the 'Download' button.
